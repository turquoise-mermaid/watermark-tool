PACKAGING GUIDE FOR WINDOWS DISTRIBUTION
=========================================

This guide explains how to package the Python tool as a Windows .exe file
that your Fiverr testers and customers can run.


PREREQUISITES
-------------
You need:
1. A Windows machine OR a Windows virtual machine/cloud instance
2. Python 3.8+ installed
3. The watermark_tool folder with all files


PACKAGING STEPS
===============

STEP 1: Install PyInstaller
----------------------------
Open Command Prompt or PowerShell and run:

    pip install pyinstaller


STEP 2: Navigate to Project Folder
-----------------------------------
    cd C:\path\to\watermark_tool


STEP 3: Create the Executable
------------------------------
Run this command:

    pyinstaller --onefile --windowed --name "WatermarkTool" ^
                --add-data "config.json;." ^
                --icon=icon.ico ^
                watermark_tool.py

OR if you don't have an icon file, use this simpler version:

    pyinstaller --onefile --windowed --name "WatermarkTool" watermark_tool.py

Explanation of flags:
- --onefile: Creates single .exe file (easier distribution)
- --windowed: Hides console window (cleaner for users)
- --name: Output filename
- --add-data: Includes additional files (optional)
- --icon: Custom icon (optional, requires .ico file)


STEP 4: Find Your Executable
-----------------------------
After PyInstaller completes (takes 1-2 minutes):

TROUBLESHOOTING PYINSTALLER
============================

Issue: "ModuleNotFoundError" when running .exe
Solution: Install missing module and rebuild:
    pip install [missing_module]
    pyinstaller --onefile --windowed watermark_tool.py

Issue: .exe is too large (>100MB)
Solution: Use virtual environment to reduce dependencies:
    python -m venv venv
    venv\Scripts\activate
    pip install -r requirements.txt
    pip install pyinstaller
    pyinstaller --onefile --windowed watermark_tool.py

Issue: Antivirus flags .exe as suspicious
Solution: This is common with PyInstaller. Solutions:
    - Code-sign the executable (requires certificate ~$100/year)
    - Upload to VirusTotal and share results with users
    - Include hash (SHA256) in documentation for verification
    - Warn users in README that antivirus may flag it

Issue: tkinter file dialogs don't work in .exe
Solution: Should work with current build. If not, try:
    pyinstaller --onefile --windowed --collect-all tkinter watermark_tool.py


ALTERNATIVE: BUILD ON LINUX FOR WINDOWS
========================================

If you ONLY have Linux (no Windows access), you can use Wine + PyInstaller:

STEP 1: Install Wine
    sudo apt install wine wine32 wine64

STEP 2: Install Python in Wine
    wine python-3.9.0.exe  (download Windows Python installer first)

STEP 3: Install PyInstaller in Wine
    wine pip install pyinstaller

STEP 4: Build
    wine pyinstaller --onefile --windowed watermark_tool.py

SECURITY BEST PRACTICES
========================

Before distributing:
[ ] Remove any hardcoded credentials/API keys
[ ] Test on fresh Windows machine
[ ] Verify no personal data in config files
[ ] Run antivirus scan on final .exe
[ ] Generate SHA256 hash for verification
[ ] Consider code-signing (optional, costs money)

To generate SHA256 hash (Windows):
    certutil -hashfile WatermarkTool.exe SHA256

Include this hash in your README for users to verify authenticity.


FILE SIZE EXPECTATIONS
======================

Expected .exe size: 15-30MB
(Includes Python interpreter + all libraries)

If larger than 50MB, something is wrong. Check:
- Unnecessary dependencies included
- Large data files bundled accidentally
- Multiple Python installations detected
